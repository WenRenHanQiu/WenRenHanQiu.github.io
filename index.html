设计模式整合详解

1，工厂模式：
简介：使用工厂模式，可以将实例对象的创建从客户端代码中抽离出来，使客户避免接触核心代码，降低了耦合度。工厂模式中所有实例对象的创建将由工厂类负责（不同种类的工厂模式的工厂类写法也不同），客户端中用户将使用工厂类创建对象，在工厂模式中，工厂类用于提供给用户接口，原用于创建对象的类
（1）简单工厂模式：
工厂模式在不同条件下可以创建不同的实体，即实现多态。
代码：
public interface Shape { //所有产品类的公共接口
void draw();
}
public class Rectangle implements Shape { //第一个产品类

@Override
public void draw() {
System.out.println(“Inside Rectangle::draw() method.”);
}
}
public class Square implements Shape { //第二个产品类

@Override
public void draw() {
System.out.println(“Inside Square::draw() method.”);
}
}
public class Circle implements Shape { //第三个产品类

@Override
public void draw() {
System.out.println(“Inside Circle::draw() method.”);
}
}
public class ShapeFactory { //工厂类（根据不同的参数创建不同的实例）

//使用 getShape 方法获取形状类型的对象
public Shape getShape(String shapeType){ //简单工厂模式中，工厂类只有一个，写法是让用户输入指定种类的参数，用switch-case语句来确定创建的对象（产品）实例
if(shapeType == null){
return null;
}
if(shapeType.equalsIgnoreCase(“CIRCLE”)){
return new Circle(); //工厂类创建产品时直接return new对象，而非继承，符合合成复用原则
} else if(shapeType.equalsIgnoreCase(“RECTANGLE”)){
return new Rectangle();
} else if(shapeType.equalsIgnoreCase(“SQUARE”)){
return new Square();
}
return null;
}
}
从以上代码中我们可以看出，工厂模式的实现步骤为：1，创建公共接口。2，创建各个实例类，均继承公共接口。3，创建工厂类，实现根据不同参数创建不同实例（多态）。
简单工厂模式优点：将公共接口，实例类，工厂类分开，提高了内聚，降低了耦合。
简单工厂模式缺点：当需要添加新产品时需要修改工厂类（违背开闭原则），代码灵活性差。

（2）工厂方法模式：
工厂方法模式：将简单工厂模式中的工厂类分为一个公共工厂接口和多个具体工厂类，所有具体工厂类均继承工厂接口，每个具体工厂类负责创建一种产品（即细化了原工厂类的功能，更符合单一职责原则）。

python爬虫

爬虫是一种获取网络上数据的方法。
在网络上获取数据有两种方法：
1，浏览器提交请求—>下载网页代码—>解析成页面
2，模拟浏览器发送请求(获取网页代码)->提取有用的数据->存放于数据库或文件中
爬虫使用的是方法二，具体步骤是：发起请求->获取相应内容->解析内容->保存数据
1，发起请求：使用http库向目标站点发起请求，即发送一个Request，请求方式有get，post等等，其中Request包含请求头和请求体，但是Request模块无法执行JS和CSS。
2，获取相应内容：请求发出后，若服务器正常响应，则可以获得一个Response，其中包括：html，json，图片，视频。
3，解析内容：解析html：解析为正则表达式（RE），或使用第三方解析工具。
解析json：解析为json模块。
解析二进制数据：以wb形式写入文件。
4，保存数据：将获取的数据存入数据库（MySQL，Redis等等）或存入文件。
其中每一步的常用工具为：
请求库：requests：较为方便，使用简单，安装方便，缺点是无法请求CSS和JS这些动态加载的信息。
selenium：使用起来较为复杂，但是可以驱动浏览器解析渲染CSS和JS，但有性能劣势（有用没用的网页都会加载）。
解析库：re（使用正则表达式）：使用方式直接简单，速度快效率高，但是逻辑较复杂，正则表达式使用较困难。
beautifulsoup：无需使用正则表达式，但使用方式较复杂（参数较多）。
pyquery：python独有，适合擅长前端的人使用。
存储库：文件，MySQL，Mongodb，Redis
附：python内置urllib库，可用于发送请求和解析（即该库包含请求库和解析库的功能）。

爬虫概念：一种按照一定的规则，自动地抓取万维网信息的程序或者脚本。
爬取的对象：浏览器通过解析前端页面代码生成我们所看到的页面，而爬虫要爬取的就是原始的前端页面代码。

操作给定的二叉树，将其变换为源二叉树的镜像。
输入描述:
二叉树的镜像定义：源二叉树
8
/ \
6 10
/ \ / \
5 7 9 11

11 9 7 5
算法：
class TreeNode {
int val = 0;
TreeNode left = null;
TreeNode right = null;

    this.val = val;

}

}
public class Solution {
public void Mirror(TreeNode root) {
TreeNode r=new TreeNode(0);
if(root!=null){
if(root.left!=null||root.right!=null){
r=root.left;
root.left=root.right;
root.right=r;
}
Mirror(root.left);
Mirror(root.right);
}
}

java各种集合的扩容机制

首先，java中并非所有集合均有扩容机制（如LinkedList因其底层结构为链表，扩容时会不断建立新节点），只有部分集合拥有系统的扩容机制，且不同的集合的扩容机制也不同，接下来对于集合的各种扩容机制进行介绍：
1，ArrayList：
因其底层结构为数组，如果像LinkedList一样逐个开辟存储空间效率会很低，故其设立了特有的扩容机制，下面是其源码：
private static final int MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8;
private void grow(int minCapacity) {
// overflow-conscious code
int oldCapacity = elementData.length;
int newCapacity = oldCapacity + (oldCapacity >> 1);
if (newCapacity - minCapacity < 0)
newCapacity = minCapacity;
if (newCapacity - MAX_ARRAY_SIZE > 0)
newCapacity = hugeCapacity(minCapacity);
// minCapacity is usually close to size, so this is a win:
elementData = Arrays.copyOf(elementData, newCapacity);
}

private static int hugeCapacity(int minCapacity) {
if (minCapacity < 0) // overflow
throw new OutOfMemoryError();
return (minCapacity > MAX_ARRAY_SIZE) ?
Integer.MAX_VALUE :
MAX_ARRAY_SIZE;
}
附：Integer.MAX_VALUE的定义：Integer.MAX_VALUE
注：ArrayList中两个属性与其容量相关：size和elementData.length，size代表当前集合中的元素个数，而elementData.length则代表当前集合的真正存储空间，而扩容就是当真正存储空间无法存储集合中所有的元素时，开辟新的存储空间再将原存储空间的数据添加进去。
从源代码中我们可以看出，在存储空间不足时，首先ArrayList会将其容量扩大至原先的1.5倍，再判断新的存储空间容量是否足够，若足够，则ArrayList的容量就是原先的1.5倍；若不够，则将其容量变为能够存储所有元素的最小存储空间，再判断当前容量（即刚好能够存储所有元素的空间）是否大于ArrayList的允许的最大存储空间（即MAX_ARRAY_SIZE = Integer.MAX_VALUE - 8），若大于，则将容量变为Integer.MAX_VALUE

java的引用传递和值传递以及String常量池

首先，从整体来说，因为基本数据类型变量所占据的存储空间所存储的是需要存储的实际值，而引用数据类型所存储的是指向需要存储实际值的存
储空间的地址，所以基本数据类型之间使用值传递，引用数据类型之间使用引用传递，但String类型和数组是例外。
先说明一下普通的值传递和引用传递原理：使用值传递的形参和实参均直接存储实际需要存储或修改的值，而形参和实参占用的是两份不同的存储
空间，且形参的存储空间在其所在的方法执行完毕后会被释放，故而只是形参存储的值被改变后其存储空间又被释放而已，实参存储的值不会改变
；而使用引用传递的形参和实参内存储的是需要存储或修改的值的存储空间的地址，即形参和实参虽然各自使用不同的存储空间，但均指向同一份
存储了实际值的存储空间，故而在方法执行完毕之后，二者所指向的同一份存储空间所存储的值也被改变了，即使形参存储空间被释放，实参所存
储（指向）的值也被改变了。
但是，String类型本身含有常量池，即含有所有字符串的池，常量池中的存储的所有值不会被改变，变量只能改变指向方向，所以除非使用new指令
开辟额外的存储空间，否则的话String类型变量会指向常量池中与存储值相同的，本来就存在的那份存储空间。而在方法执行完毕后，只是形参的
指向（存储的地址）改变了，故而实参所存储（指向）的值仍未改变。
数组实参传递给形参的是数组名，即指向数组实际存储值存储空间的地址，故其原理和普通引用传递是相同的，所以无论是引用数据类型数组还是
基本数据类型数组均使用引用传递。

29,递归

简单来说，递归就是方法自己调用自己，但是需要终止条件来防止造成死循环。
相关概念：直接递归，间接递归，尾递归，线性递归和树状递归。
直接递归即为在方法的return中直接（显式）地调用自己。
间接递归即为方法在return语句中未调用自己，而是调用了其他方法，再由其他方法来调用自己（隐式调用）。
尾递归：即在方法的return语句中仅调用自己，无任何其他计算（未决操作）；尾递归的优点在于其每次递归不需要存储未决操作的相关信息，即需
存储的信息量不会随递归次数的增加而增加，可以节省大量资源。一般将普通递归转化为尾递归的方法是将表达式中其他的参数作为方法的参数。
线性和树状递归：线性递归指未决操作中不含递归调用，即每次递归只会产生一个调用；而树状递归则在未决操作中含有递归调用，即每次调用会分
支产生多个递归调用。

28,迭代器

java的集合种类众多，它们分别有着不同的底层数据结构，故而它们各自的遍历方式也不同，为了让用户无需了解集合的底层结构也能便捷地遍历集
合，java引入了迭代器。
迭代器即为Iterator接口的对象，根据遍历的集合类型不同，实例化时所使用的构造方法也不同。
通常使用迭代器的格式为：
List list = new ArrayList();
Iterator listIt = list.iterator();
while(listIt.hasNext()){
System.out.println(listIt.next());
}
注：因为ArrayList类间接继承了Iterator接口，重写了iterator()方法，故list.iterator()方法返回的是实例，使Iterator对象listIt可以被实例
化。
Iterator接口中常用的只有hasNext()和next()两个方法，hasNext()返回boolean型值，判断集合中是否还有元素；next()返回集合中的元素，并将
索引指向下一个元素，使下一次调用next()方法时遍历到下一个元素。
注：直接使用（即可实例化）的集合类（如ArrayList，LinkedList等）均为间接继承Iterator接口，故这些集合类中未重写hasNext()方法和next()
方法，以ArrayList类为例，该类继承了AbstractList类，在AbstractList类中才重写了这两个方法，以下是源代码：
public boolean hasNext() {
return cursor != size();
}
public E next() {
checkForComodification();

int i = cursor;
E next = get(i);
lastRet = i;

}
当然，对于不同的集合类，重写的hasNext()和next()也不同。
注：使用迭代器遍历时不允许对集合元素进行增删（即改变集合的size），否则会产生异常；但是可以改变元素，不会产生异常。
