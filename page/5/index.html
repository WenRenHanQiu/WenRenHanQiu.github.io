1,Abstract与哪些类不能共存，为什么

Abstract关键字的含义为抽象，以该关键字修饰的方法为抽象方法，抽象方法无方法体；以该关键字修饰的类为抽象类，其中必须含有
抽象方法。private含义为私有，final含义为不可更改，其修饰常量和不可重写方法；static含义为静态，修饰静态变量与静态方法，静态
变量及方法归类所有，而非对象。final与Abstract不能共存，因为抽象方法（或类）是为了重写（或派生）以实现多态，而final却禁止方
法被重写（或被继承）二者冲突。

0,面向对象三大特点及其含义

面向对象三大特性：封装，继承，多态。封装：即将所有方法与属性均封装起来，不对用户显示，仅留给用户接口以进行输入和输出，
可以有效地保护代码不被修改，保证其安全性；继承即类（接口）之间的继承，可以使类依据不同功能要求去进行派生，可以实现重用代码
，简化程序；多态：即同一个方法，面对不同的出入，会有不同的操作或输出；或同一操作，对于不同的对象，有不同的解释，产生不同的
结果。

HashMap分享心得

HashMap的扩容机制：实质上是开辟一个新的数组再将原数组复制到新数组内。需要注意的是，哈希算法得到的哈希地址与哈希桶数组的大小有关，
所以在扩容之后由于哈希桶数组大小变化（变为原来的两倍），所以需要对每一个元素进行再哈希，重新存储在哈希桶数组里。而且即使能扩容，
也不允许容量超过MAXIMUM_CAPACITY（为整型静态常量，大小为1 << 30=1073741824）。
HashMap的负载因子：HashMap具有扩容机制，但并非是在其所含元素超过最大容量时才会进行扩容，而是在所含元素数量超过最大容量与负载因子
的乘积时便进行扩容，所以，负载因子必小于1。负载因子的意义：有序数组存储数据，对数据的索引效率都很高，但是插入和删除就会有性能瓶颈
，链表存储数据，要一次比较元素来检索出数据，所以索引效率低，但是插入和删除效率高，两者取长补短就产生了哈希散列这种存储方式，也就
是HashMap的存储逻辑：负载因子表示一个散列表的空间的使用程度，有这样一个公式：initailCapacity*loadFactor=HashMap的容量。所以负载因
子越大则散列表的装填程度越高，也就是能容纳更多的元素，元素多了，链表大了，所以此时索引效率就会降低。反之，负载因子越小则链表中的
数据量就越稀疏，此时会对空间造成浪费，但是此时索引效率高。
HashMap的重要方法：
1，put方法：首先判断键值对数组是否为空或null，如果是，则扩容（resize（）方法）；再以键值key计算hash地址得到其数组索引i（即所存储
到的数组的下标），若table[i]为null，则直接存储，若不为null，则查看已被存储的元素的key值，若相同，则覆盖其value（相当于执行了改变
元素值的操作），若不同，则检查table[i]是否为treeNode（即table[i]是否为红黑树结构（jdk1.8新特性）），若是，则直接在红黑树中插入键
值对存储，若不是（则table[i]必为链表结构），遍历table[i]链表长度是否大于8（算上即将被添加的这个元素），若是，则将其结构转化为红
黑树再进行存储，否则，直接在该链表上插入存储，插入时若遍历到相同的key值，则直接覆盖value（与上同）；插入成功后，再判断此时的键值
对数量size是否超过了最大容量threshold，若超过，则进行扩容。
附：关于threshold（以下为jdk1.8的新特性）：HashMap含有TREEIFY_THRESHOLD和UNTREEIFY_THRESHOLD，二者均为静态常量，其中
TREEIFY_THRESHOLD的值固定为8，其作用为作为table[i]将链表结构变为红黑树的阈值；UNTREEIFY_THRESHOLD值固定为6，其作用为作为table[i]
的数据结构由红黑树变为链表的阈值。（即当table[i]中所含元素为7时，无法确定其结构为红黑树还是链表）

哈希冲突：以哈希算法对与不同的存储对象算出了相同的哈希地址，使得在存储时产生冲突。
处理哈希冲突的四种方法：
1，开放定址法：此法是将产生的冲突的哈希地址添加增量之后再对哈希表长度使用除留余法，直到计算出无冲突的哈希地址。
而增量添加的方式有两种：线性探测再散列，平方探测再散列；即每次添加的增量为：1，2，3……和1的平方，负的1的平方，二的平方，负的二
的平方，三的平方，负的三的平方……
2，链地址法（java中HashMap使用该方法）：将所有哈希地址相同的元素连接为一个链表，哈希表存放链表首元素的地址。(到了jdk1.8，当同一个
hash值的节点数不小于8时，不再采用单链表形式存储，而是采用红黑树)
3，再哈希法：用其他的哈希函数去计算哈希地址，直到无冲突。
4，建立公共溢出区：将所有冲突的元素存放在另一个区域，将其称之为公共溢出区。

抽奖问题

抽奖问题：简单来说就是已知奖券种类数（m）和买的奖券张数（n），奖券可重复，集齐所有种类奖券即中奖，求中奖的概率。
解决该题的思想是动态规划，设已买奖券数为i，奖券种类数为m，已买奖券中种类数为j，中奖概率为f（i，j），则中奖概率的递推公式为：
f（i，j）=f（i-1，j）j/m+f（i-1，j-1）（m-j+1）/m，意义为：买i张奖券集齐j种奖券的概率为：买i-1张奖券集齐j种奖券的概率乘以j/m
，加买i-1张奖券集齐j-1种奖券的概率乘以（m-j+1）/m，即买i张奖券集齐j种奖券的概率等于买第i张奖券集齐第j种奖券的概率，加买第i张
奖券是之前已经有的j种奖券（即重复）的概率。则当j（已集齐的奖券数）等于m（奖券种类总数）时，f（i，j）即为所求解。

Q：为什么买第i张奖券获得新奖券的概率与未获得新奖券的概率之和不是1呢？
A：因为本题目求的是买n张奖券中奖（即及其m种图案）的概率，而本题的解题思路是：先求买1张奖券中奖的概率，再求买2张奖券中奖的概率，
直到求出买n张奖券中奖的概率；其中每一次计算概率的方式是：计算买i张奖券集齐j种图案的概率是买第i张奖券获得新图案和未获得新图案
的概率之和，因为买完第i张奖券后集齐的奖券种类数为j，故计算买第i张奖券获得新图案的概率是(m-j+1)/m（在买第i张奖券之前已获得j-1
种奖券），否则概率是j/m（在买第i张奖券之前已获得j种奖券）。故二者之和不为1。

特点：ArrayList可以动态增长和缩减（所有集合的共性，也是集合相比于数组的最大区别），底层为数组，线程不安全。
动态增长与缩减的原因：该类封装了一个动态再分配的Object[]数组，使每个类对象均含有一个名为capacity的属性，该属性表示并控制数组的长度，比如：当向ArrayList内添加元素时，该属性值便会增加。
与数组的length不同，ArrayList含有size和Capacity两个属性，数组的length属性即为数组定义时的大小，与数组中的对象个数无关，但ArrayList的size为集合内对象的个数，Capacity为集合的存储空间大小。

快速排序程序分析

package com.google.javase.zs;
//快速排序
public class QuickSore{
public static void quickSore(int[] data, int left, int right) { //该方法的递归即为分治法
int base;
if(right>left) { //if内的条件即为递归中止条件
base=sort(data,left,right);
quickSore(data,left,base-1);//小于基准
quickSore(data,base+1,right);//大于基准
}
}
public static int sort(int[] data, int left, int right) { //sort函数即为在一个基准下的排序
int zj=data[left];
int base=left,zjz; //普通快排为选取数组第一个元素的值为基准值
data[left]=data[right]; //此两步将原数组的首元素与尾元素的值互换
data[right]=zj;
for(int i=left;i<right;++i) {
if(data[right]>data[i]) { //该if语句等价于以末尾值为基准值进行快排
zjz=data[base];
data[base]=data[i];
data[i]=zjz;
base++;
}
}
zjz=data[base]; //此处zjz的值为数组中小于等于末尾值的所有值中序号最大的元素的值
data[base]=data[right];
data[right]=zjz; //此处交换data【base】和data【right】的值之后，data【base】之前的所有值均小于data【base】，其之后的所有值均大于data【base】
return base;
}
public static void main(String[] args) {
int []data= {3,4,1,2,6,4,5,7,8,0,9,43,98,88,5,7};
quickSore(data,0,data.length-1);
for(int i=0;i<data.length;++i) {
System.out.print(data[i]+” “);
}
System.out.println();
}
}
