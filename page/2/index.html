java的引用传递和值传递以及String常量池

首先，从整体来说，因为基本数据类型变量所占据的存储空间所存储的是需要存储的实际值，而引用数据类型所存储的是指向需要存储实际值的存
储空间的地址，所以基本数据类型之间使用值传递，引用数据类型之间使用引用传递，但String类型和数组是例外。
先说明一下普通的值传递和引用传递原理：使用值传递的形参和实参均直接存储实际需要存储或修改的值，而形参和实参占用的是两份不同的存储
空间，且形参的存储空间在其所在的方法执行完毕后会被释放，故而只是形参存储的值被改变后其存储空间又被释放而已，实参存储的值不会改变
；而使用引用传递的形参和实参内存储的是需要存储或修改的值的存储空间的地址，即形参和实参虽然各自使用不同的存储空间，但均指向同一份
存储了实际值的存储空间，故而在方法执行完毕之后，二者所指向的同一份存储空间所存储的值也被改变了，即使形参存储空间被释放，实参所存
储（指向）的值也被改变了。
但是，String类型本身含有常量池，即含有所有字符串的池，常量池中的存储的所有值不会被改变，变量只能改变指向方向，所以除非使用new指令
开辟额外的存储空间，否则的话String类型变量会指向常量池中与存储值相同的，本来就存在的那份存储空间。而在方法执行完毕后，只是形参的
指向（存储的地址）改变了，故而实参所存储（指向）的值仍未改变。
数组实参传递给形参的是数组名，即指向数组实际存储值存储空间的地址，故其原理和普通引用传递是相同的，所以无论是引用数据类型数组还是
基本数据类型数组均使用引用传递。

29,递归

简单来说，递归就是方法自己调用自己，但是需要终止条件来防止造成死循环。
相关概念：直接递归，间接递归，尾递归，线性递归和树状递归。
直接递归即为在方法的return中直接（显式）地调用自己。
间接递归即为方法在return语句中未调用自己，而是调用了其他方法，再由其他方法来调用自己（隐式调用）。
尾递归：即在方法的return语句中仅调用自己，无任何其他计算（未决操作）；尾递归的优点在于其每次递归不需要存储未决操作的相关信息，即需
存储的信息量不会随递归次数的增加而增加，可以节省大量资源。一般将普通递归转化为尾递归的方法是将表达式中其他的参数作为方法的参数。
线性和树状递归：线性递归指未决操作中不含递归调用，即每次递归只会产生一个调用；而树状递归则在未决操作中含有递归调用，即每次调用会分
支产生多个递归调用。

28,迭代器

java的集合种类众多，它们分别有着不同的底层数据结构，故而它们各自的遍历方式也不同，为了让用户无需了解集合的底层结构也能便捷地遍历集
合，java引入了迭代器。
迭代器即为Iterator接口的对象，根据遍历的集合类型不同，实例化时所使用的构造方法也不同。
通常使用迭代器的格式为：
List list = new ArrayList();
Iterator listIt = list.iterator();
while(listIt.hasNext()){
System.out.println(listIt.next());
}
注：因为ArrayList类间接继承了Iterator接口，重写了iterator()方法，故list.iterator()方法返回的是实例，使Iterator对象listIt可以被实例
化。
Iterator接口中常用的只有hasNext()和next()两个方法，hasNext()返回boolean型值，判断集合中是否还有元素；next()返回集合中的元素，并将
索引指向下一个元素，使下一次调用next()方法时遍历到下一个元素。
注：直接使用（即可实例化）的集合类（如ArrayList，LinkedList等）均为间接继承Iterator接口，故这些集合类中未重写hasNext()方法和next()
方法，以ArrayList类为例，该类继承了AbstractList类，在AbstractList类中才重写了这两个方法，以下是源代码：
public boolean hasNext() {
return cursor != size();
}
public E next() {
checkForComodification();
try {
int i = cursor;
E next = get(i);
lastRet = i;
cursor = i + 1;
return next;
} catch (IndexOutOfBoundsException e) {
checkForComodification();
throw new NoSuchElementException();
}
}
当然，对于不同的集合类，重写的hasNext()和next()也不同。
注：使用迭代器遍历时不允许对集合元素进行增删（即改变集合的size），否则会产生异常；但是可以改变元素，不会产生异常。

27,基本数据类和包装类（自动拆装箱）

装箱即为将基本数据类型转化为其对应的引用类型，拆箱即为将特定的某些引用类型转化为其对应的基本数据类型。
java中每种基本数据类型均对应有包装器类型（引用类型），而一般引用类型在使用格式为ArrayList list=new ArraayList();而ArrayList list的
形式会造成空指针异常，但在定义Integer类型变量时我们一般会写：Integer i=10；并没有用new指令（JDK5新特性），这就是自动装箱。同样的道
理，Integer类型变量i在执行i=i+5；语句时，5为int型，需要将i拆箱为int型进行计算，再将其装箱为Integer类型，该过程中拆装箱均不需要用户
写相关代码，这就是自动拆装箱。
自动拆装箱优点：增强了代码的可读性，简化了代码。
注：当数值在byte范围内(-128-127)时，自动装箱时不会新创建对象。（即数值相同的两个Integer型变量使用==运算符的返回值为true）
基本数据类（8种）及其存储空间：int（4字节）byte（1字节）short（2字节）long（8字节）float（4字节）double（8字节）char（2字节）
boolean（不确定）
注：（1）如果书写long型的值，则需要在数值后面添加字母L，大小写均可以。
（2）小数型的取值范围以科学计数形式进行描述。
（3）float类型的小数，需要在小数后加字母F，不区分大小写，例如4.09f或4.09F，但double类型赋小数值时不必加f，可以直接写。
（4）布尔型占用的空间取决于Java虚拟机（JVM）的实现，可能是1bit，也可能是8bit，且其默认值为false。
（5）基本数据类未包装任何方法及特殊属性，仅包含数据。
包装类：即包装了除数据外的其他属性和方法的类。除了8种基本数据类型以外，其他数据类型均为包装类（引用类型）特殊的，Integer，Byte，
Short，Long，Boolean，Character，Float，Double这些引用类型可以拆箱为基本数据类型。

26,线程和进程

在操作系统中，每一个独立运行的程序都是一个进程，而进程可以包含一个或多个线程，而多线程，就是在一个进程中，开启多个线程，使多个线程
交替执行，由于CPU运行处理很快，所以看起来似乎在同时处理多个线程，CPU执行进程也是如此。
而二者的区别在于资源管理方式不同，每个进程都有独立的地址空间，所以一个进程崩溃后，其他线程不会受到影响；但是各个线程之间并没有单独
的地址空间，一个线程崩溃就会使整个进程崩溃。不过也正因如此，进程无法共享变量实现并发，但线程可以。
java中实现多线程有两种方式：一种是继承Thread类并重写Thread类的run()方法，重写的run()就是该线程运行的代码；另一种是继承Runnable接口
，也是run()方法就是线程执行的代码。二者的不同之处在于，使用继承Thread类的方式无法实现多个线程共同抢夺临界资源，但是使用继承Runnabl
e接口的方式则可以。
线程有五种状态：新建状态，就绪状态，运行状态，阻塞状态，死亡状态。
新建状态：线程在刚被创建时，会短暂地处于这种状态，此时线程已被分配系统资源并完成了初始化，有资格获得CPU运行时间。
就绪状态：线程在执行start()方法之后就会进入这种状态，此时只要线程得到CPU时间就会进入运行状态。
运行状态：就是线程运行时的状态，可能被阻塞也可能执行完毕进入死亡状态。
阻塞状态：运行状态的进程如果遇到以下几种情况就会进入阻塞状态：调用了wait()，join()或sleep()方法，调用了IO阻塞方法，等待执行上锁的
代码块或方法。阻塞状态在以下情况下会进入就绪状态：获得了同步锁，阻塞IO方法返回，sleep()方法等待时间到，调用notify()或notifyAll()方
法，调用join()方法的线程终止。
死亡状态：运行状态的线程执行完毕或产生Exception或Error会进入死亡状态。
相关方法说明：sleep()方法：以数字作为参数，代表等待的毫秒数，使线程在规定的时间内进入阻塞状态，期间线程的锁不释放，且notify()或
notifyAll()无法将线程唤醒。
wait()方法：有两种使用方式：第一种的作用和sleep()方法相似，但是期间线程的锁释放，而且可以被notify()或notifyAll()唤醒；第二种方式无
参数，此时线程将会无线地等待下去，除非被notify()或notifyAll()唤醒。
线程在被调度时会参考线程的优先级（由线程的优先级参数决定），但是仍有一定的随机性，哪些线程被分配CPU时间是无法预测的。

25,泛型

集合中的各个元素的类型可以不同，如果想要保证集合中所有元素均为某一种类型，就需要用到泛型。
集合不会记住所加入对象的类型，编译时所有元素为Object类型，但元素在被取出调用时其仍然为其原本的类型，且其原本类型的特性并未改变。为
了让集合“记住”集合内元素的类型，需要用到泛型。
泛型使用格式：集合类型<基本类型> 集合名=new 集合类型<基本类型>()
使用泛型之后，在编译时编译器就会认定集合中所有的元素类型必须为指定的类型，否则就会报错。

24,异常及异常的注意事项

使用异常，是为了处理在运行时可能出现的错误，try块测试可能出异常的代码，catch块在捕获异常后进行处理，使用户可以自己控制异常的处理，
而且用户可以自行确定捕获的异常的类型和范围，使系统不必去检查是哪个特定错误；而且，一个try块可以对应多个catch块，使用户可以对于异常
的捕获和处理更为精确，不过也因此需要注意：
try{}
catch(Exception e){}
catch(Exception e){}
catch语句是按顺序执行的，所以catch中的异常参数的类型范围需要从小到大。
使用异常，还可以使用户更便捷地记录异常，以便用户对异常进行分类和处理。
除了已有的异常类，还可以自定义异常类，不过必须继承已有的异常类。
异常机制包含异常说明，可以在不泄露源代码的前提下对于发生的异常进行说明。
在try块和catch块之后还可以添加finally块，在finally块中的语句在执行完try和catch块之后必然会被执行，无论是否有异常抛出（即使catch或
try块内含有return语句，finally块也会在return语句执行前被执行）。
对于异常的处理，有两种方式，第一种是直接处理，即对于产生的问题作出相应地处理；还有一种方式是将该异常向上抛出，但是使用向上抛出时需
要注意，当异常已经出现在主方法时，不能再向上抛出，因为主方法已经是最顶层了。
附：java中常见的异常类型：NullPointerException 空指针异常 ClassNotFoundException 指定类不存在异常 NumberFormatException 字符串转数
字异常 IndexOutOfBoundsException 数组下标越界异常 FileNotFoundException 文件未找到异常 ClassCastException 数据类型转换异常 Illegal
ArgumentException 方法参数错误异常 IllegalAccessException 无访问权限异常 ArithmeticException 数学运算异常。

23,throw和throws

throw和throws均为java中抛出异常形式；java中共有三种抛出异常的形式：throw，throws和系统自动抛出异常。其中系统抛出异常为自动（如：
int a = 5, b =0; System.out.println(5/b);语句在编译时不会报错，但在程序运行时会由系统自动抛出异常），throw和throws抛出异常需要人
为控制。
throw和throws的主要区别在于：throw是直接抛出异常（格式：throw 异常对象），执行该语句即会抛出异常；而throws用于声明某方法可能会抛出
异常（格式：throws 异常类型），写在方法的参数列表后。故而，throw出现在方法体中，throws出现在方法头中。
throw指令意为“向上抛出”，即在其上层中需要有catch块来捕捉该异常，故throw不能用于主方法中（因为主方法无上层）；throws修饰的方法在调
用时需要将其放入try块中，在出现异常时可以用catch块捕捉。
