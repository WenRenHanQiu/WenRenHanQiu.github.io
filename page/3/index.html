20,Collection和Collections的区别

首先，二者最大的区别在于Collection是接口，而Collections是实体类。
map，set，list，vector均实现了Collection接口，故它们的子类，那些常用的集合类均继承了Collextion接口，但是那些常用的集合类也包装了
Collections的一些方法，故Collection是所用集合类的接口，而Collections是为那些集合类提供方法的辅助类。
如：ArrayList中的size()，isEmpty()，add()等方法继承自Collection，而它的sort方法则调用Collections的sort方法。

19,传统for和增强for的区别

增强for相比于传统for的优点是更容易遍历，在仅遍历集合元素的情况下使用增强for循环效率更高，但是增强for循环无法使用循环变
量对集合元素进行修改，而传统for循环却可以。
传统for循环格式：
for(int i=0;i<n;i++){ //注意：n为与i相同类型的变量或常量
//循环体
}
增强for循环格式：
for(int i:n){ //注意：n为集合或数组名；i每次循环时的值为n中元素的值，要求i与n中元素类型相同
//循环体 //由于集合中存入的元素类型都会被系统默认为Object类型，故若要以特定类型遍历集合需要使用泛型
}

14,String和StringBuffer的区别

String和StringBuffer的主要区别为StringBuffer比String类型多了缓冲区，其效率比String更高。
String类型创建的对象为字符串常量，而StringBuffer创建的是变量，所以每次对于String类型改变都相当于生成了一个新的对象，但对StringBuff
er类型对象的改变是改变原对象的，不会生成新对象。所以一般推荐使用StringBuffer类，尤其是对象经常改变时。

12,数组和集合的区别

数组是一种数据结构，也是部分集合的底层数据结构，但是其本身不像集合一样自带方法（比如查询长度，自动扩容等），数组的优点是
便于查找，但不便于增删；集合多种多样，底层数据结构有数组也有链表，可以根据不同的需要进行选择，而且集合拥有多种方法，便于使
用，部分集合是线程安全的，可以确保信息安全。集合和数组的最大区别是数组的大小必须在定义时就确定且不可更改，但是集合的大小无
需在定义时确定，其长度为默认长度，在存储空间不足时集合还可以自动扩容，非常方便。

11,同步方法与同步代码块的区别

java中实现线程同步的手段即使用同步方法或同步代码块。同步方法和同步代码块即为以synchronized关键字修饰的方法和代码块。使用该关键字修
饰的方法或代码块在同一时间内只能由一个线程调用，若此时其他线程要调用该方法或代码块，该线程会被阻塞。
而二者的不同之处在于，上锁的部分不同，同步方法是对某方法上锁，而同步代码块是对于某代码块上锁，相对于同步方法而言，同步代码块造成阻
塞的可能性更小。比如说，在某线程调用同步方法时，其他要调用该同步方法的线程均会阻塞，但是，若只是将方法中的部分代码块上锁，则其他要
调用该方法的线程仍然可以调用该方法中的非同步代码块，从而减少了阻塞。
