设计模式整合详解

1，工厂模式：
简介：使用工厂模式，可以将实例对象的创建从客户端代码中抽离出来，使客户避免接触核心代码，降低了耦合度。工厂模式中所有实例对象的创建将由工厂类负责（不同种类的工厂模式的工厂类写法也不同），客户端中用户将使用工厂类创建对象，保证客户端中不会出现任何实际产品类的类名或构造方法。
（1）简单工厂模式：
简单工厂模式在不同条件下可以创建不同的实体，即实现多态，实现方式为直接使用if-else或switch-case来筛选。
代码：
public interface Shape { //所有产品类的公共接口
void draw();
}
public class Rectangle implements Shape { //第一个产品类

@Override
public void draw() {
System.out.println(“Inside Rectangle::draw() method.”);
}
}
public class Square implements Shape { //第二个产品类

@Override
public void draw() {
System.out.println(“Inside Square::draw() method.”);
}
}
public class Circle implements Shape { //第三个产品类

@Override
public void draw() {
System.out.println(“Inside Circle::draw() method.”);
}
}
public class ShapeFactory { //工厂类（根据不同的参数创建不同的实例）

//使用 getShape 方法获取形状类型的对象
public Shape getShape(String shapeType){ //简单工厂模式中，工厂类只有一个，写法是让用户输入指定种类的参数，用switch-case语句来确定创建的对象（产品）实例
if(shapeType == null){
return null;
}
if(shapeType.equalsIgnoreCase(“CIRCLE”)){
return new Circle(); //工厂类创建产品时直接return new对象，而非继承，符合合成复用原则
} else if(shapeType.equalsIgnoreCase(“RECTANGLE”)){
return new Rectangle();
} else if(shapeType.equalsIgnoreCase(“SQUARE”)){
return new Square();
}
return null;
}
}
客户端：
ShapeFactory s=new ShapeFactory();
Shape shape=s.getShape(“CIRCLE”);
shape.draw();
从以上代码中我们可以看出，工厂模式的实现步骤为：1，创建公共接口。2，创建各个实例类，均继承公共接口。3，创建工厂类，实现根据不同参数创建不同实例（多态）。
简单工厂模式优点：将公共接口，实例类，工厂类分开，提高了内聚，降低了耦合避免用户直接接触构造类。
简单工厂模式缺点：当需要添加新产品时需要修改工厂类（违背开闭原则），代码灵活性差。

（2）工厂方法模式：
工厂方法模式：将简单工厂模式中的工厂类分为一个公共工厂接口和多个具体工厂类，所有具体工厂类均继承工厂接口，每个具体工厂类负责创建一种产品（即将一个工厂类变为一个工厂接口与多个具体工厂类，细化了原工厂类的功能，更符合单一职责原则），
即建立一个工厂接口，让其子类自己决定创建何种对象（产品），这样需要创建新产品时只需创建新的工厂类即可，避免了在原工厂类或工厂接口上进行修改。
代码：
产品接口：
public interface ICar
{
void GetCar();
}
具体产品类：
1，跑车类：
public class SportCar implements ICar
{
public void GetCar()
{
Console.WriteLine(“生产一辆跑车。”);
}
}
2， 吉普车类：
public class JeepCar implements ICar
{
public void GetCar()
{
Console.WriteLine(“生产一辆吉普车。”);
}
}
3，掀背车类：
public class HatchbackCar implements ICar
{
public void GetCar()
{
Console.WriteLine(“生产一辆掀背车。”);
}
}
工厂接口：
public interface IFactory
{
ICar CreateCar();
}
具体工厂类：
1，跑车工厂类：
public class SportFactory implements IFactory
{
public ICar CreateCar()
{
return new SportCar();
}
}
2，吉普车工厂类：
public class JeepFactory implements IFactory
{
public ICar CreateCar()
{
return new JeepCar();
}
}
3，掀背车工厂类：
public class HatchbackFactory implements IFactory
{
public ICar CreateCar()
{
return new HatchbackCar();
}
}
客户端：
IFactory i=new JeepFactory();
ICar c=i.CreateCar();
c.GetCar();
在这里，有些同学可能有疑问：为什么之前简单工厂类中有switch进行选择创建何种对象，而工厂方法模式中的工厂类却没有选择的功能了？
这是因为，选择确认用户需要创建何种对象，本就不是工厂类的事，工厂类只需负责创建各种对象，只不过简单工厂模式的工厂类耦合程度较高，未将各种对象的创建分离（违背了单一职责原则），不得已需要进行选择。
（3）抽象工厂模式：
抽象工厂模式：以上两种工厂模式只能解决对于单一产品的生产，如果我们想要生产出产品的组合的话，就需要用到抽象工厂模式，它用于生产多种产品的组合，
简单来说，相比于工厂方法模式，抽象工厂模式就是讲一个产品接口变为多个产品接口，增加了产品的种类，工厂类所生产的产品种类也变成了多种。
抽象产品接口：
///


/// 抽象车接口
///
public interface ICar
{
///
/// 车的类型属性
///
String Type；

    void getType();
}

/// <summary>
/// 抽象背包接口
/// </summary>
public interface IBackpack
{
    /// <summary>
    /// 包的颜色属性
    /// </summary>
    String Color;

    void getColor();
}

具体产品类：
///


/// 跑车
///
public class SportCar implements ICar
{

    /// <summary>
    /// 重写接口的方法
    /// </summary>
    String Type="跑车";

    public void getType()
    {
        return this.Type;
    }
}

/// <summary>
/// 吉普车
/// </summary>
public class JeepCar implements ICar
{

    /// <summary>
    /// 重写接口的方法
    /// </summary>
    String Type="吉普车";

    public void getType()
    {
        return this.Type;
    }
}

/// <summary>
/// 蓝色背包
/// </summary>
public class BlueBackpack implements IBackpack
{
    /// <summary>
    /// 重写基类的Color属性
    /// </summary>
    String Color="蓝色";

    public void getColor()
    {
        return this.Color;
    }
}

/// <summary>
/// 红色背包
/// </summary>
public class RedBackpack implements IBackpack
{
    /// <summary>
    /// 重写基类的Color属性
    /// </summary>
    String Color="红色";

    public void getColor()
    {
        return this.Color;
    }
}

抽象工厂接口：
///


/// 抽象工厂接口：
///
public implements IFactory
{
///
/// 抽象方法： 创建一辆车
///
///
ICar CreateCar();

    /// <summary>
    /// 抽象方法： 创建背包
    /// </summary>
    /// <returns></returns>
    IBackpack CreateBackpack();
}

具体工厂类：
///


/// 蓝色背包与吉普车的产品组合
///
public class Factory implements IFactory
{
public ICar CreateCar()
{
return new JeepCar();
}

    public IBackpack implements CreateBackpack()
    {
        return new BlueBackpack();
    }
}

客户端：
IFactory i=new Factory();
ICar c=i.CreateCar();
IBackpack b=i.CreateBackpack();
System.out.println(c.getType());
System.out.println(b.getColor());
